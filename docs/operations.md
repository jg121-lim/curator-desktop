# Operations Guide

## 1. Windows 배포 절차

1. **빌드 환경 준비**
   - Node.js LTS, pnpm/npm, Rust toolchain(필요 시), Visual Studio Build Tools를 설치합니다.
2. **의존성 설치**
   - 루트에서 패키지 매니저로 의존성을 설치합니다.
3. **앱 빌드**
   - Desktop 앱의 프로덕션 번들을 생성합니다.
   - 코드 서명 인증서(사내 PKI 또는 상용 인증서)로 실행 파일을 서명합니다.
4. **설치 패키지 생성**
   - MSI/EXE 설치 프로그램을 생성하고 버전/릴리즈 노트를 포함합니다.
5. **검증 및 배포**
   - Windows 10/11 환경에서 설치/업데이트/삭제를 검증합니다.
   - 사내 배포 채널(Intune, SCCM, 사내 포털 등)에 업로드합니다.

## 2. 사내 API 키 관리 정책

- **저장 금지 원칙**: API 키는 코드 저장소, 평문 설정 파일, 로그에 저장하지 않습니다.
- **주입 방식**: 런타임 시 OS 시크릿 저장소(예: Windows Credential Manager) 또는 사내 시크릿 매니저에서 주입합니다.
- **최소 권한**: 환경별(dev/stage/prod)로 키를 분리하고 권한 범위를 최소화합니다.
- **회전 정책**: 정기 회전(예: 90일) 및 유출 의심 시 즉시 폐기/재발급합니다.
- **감사 추적**: 키 발급, 사용, 폐기 이력을 감사 로그에 남깁니다.

## 3. Confluence API 제한(1초 1회) 대응 전략

- **Rate Limiter 강제**: Confluence 클라이언트 공용 계층에 1RPS 토큰 버킷/리키 버킷 제한을 기본 적용합니다.
- **작업 큐 직렬화**: 증분 동기화 작업에서 API 호출을 큐잉해 병렬 요청이 제한을 넘지 않게 합니다.
- **백오프 + 재시도**: 429/5xx 응답 시 exponential backoff(지터 포함)로 재시도합니다.
- **체크포인트 기반 증분 수집**: `last_synced_at` 이후 변경분만 조회하여 총 호출량을 감소시킵니다.
- **관측성**: 초당 호출 수, 429 발생률, 평균 동기화 지연을 메트릭으로 수집하고 경보를 설정합니다.
