# Agent 역할 정의

## 1) Architecturing Agent
- 시스템 경계와 도메인 구조를 정의한다.
- 패키지/서비스 간 의존성과 인터페이스 정책을 수립한다.
- 배포 타깃(특히 Windows 우선 전략) 기준으로 기술 의사결정을 문서화한다.

## 2) Front-end Agent
- `apps/desktop`의 Electron + React UI/UX를 구현한다.
- 데스크톱 앱의 상태 관리, 라우팅, 사용자 상호작용을 담당한다.
- 백엔드 API 연동 시 에러 처리 및 사용자 피드백 규칙을 준수한다.

## 3) Back-end Agent
- `apps/backend`의 FastAPI 엔드포인트, 비즈니스 로직, 데이터 모델을 구현한다.
- API 계약(OpenAPI/스키마)을 `packages/shared`와 일치시키도록 관리한다.
- 인증, 로깅, 예외 처리 등 서버 안정성 기준을 유지한다.

## 4) Documentation Agent
- 개발/운영 문서(README, 설계 문서, 실행 가이드)를 최신 상태로 유지한다.
- 신규 기능 반영 시 변경점과 마이그레이션 절차를 문서화한다.
- 팀 온보딩에 필요한 최소 실행 절차를 명확히 제공한다.

## 5) Test Agent
- 단위/통합/E2E 테스트 전략을 수립하고 자동화한다.
- CI에서 실행 가능한 품질 게이트(테스트, 린트, 타입체크)를 정의한다.
- 릴리스 전 회귀 테스트 범위를 점검하고 결과를 공유한다.
